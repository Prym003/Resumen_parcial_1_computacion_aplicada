Unidad 1 — GNU/Linux: Historia, Origen, Evolución y Virtualización

1.1 Historia y Software Libre EXAMEN

Richard Stallman (nacido en 1953, NY). Estudiante del MIT en los 70s, parte del laboratorio de IA. Vio cómo el software pasaba de cultura de compartir a software propietario (caso impresora Xerox).

  • En 1983 anunció el Proyecto GNU ("GNU's Not Unix") — un sistema operativo completamente libre.
  • Escribió la licencia GPL (General Public License) que garantiza que el software siga siendo libre.
  • Creó herramientas fundamentales: GCC (compilador) y Emacs (editor).
  • Stallman NO creó Linux — creó el proyecto GNU y promovió el software libre.

1.2 Origen de Unix → GNU → Linux

  • Años 60: MIT + GE + Bell Labs desarrollan Multics.
  • 1969: Bell Labs abandona Multics. Ken Thompson crea Unix (1971).
  • 1973: Unix se reescribe en lenguaje C.
  • 1975: aparece BSD (UC Berkeley).
  • 1980: Unix se fragmenta (System V de AT&T, SunOS, AIX, HP-UX).
  • 1983: Proyecto GNU (Stallman).
  • 1991: Linus Torvalds, estudiante finlandés, crea el kernel Linux (basado en Minix de Tenenbaum). Publica el famoso mensaje en Usenet: "Hola a todos los que están usando Minix...".
  • 2001: Apple lanza macOS basado en Unix (BSD).
⭐ Concepto clave del examen:
  • GNU = conjunto de herramientas libres (Stallman).
  • Linux = el kernel (Torvalds).
  • GNU/Linux = la combinación de ambos = el sistema operativo completo.
El kernel es el núcleo: controla el hardware, gestiona memoria y periféricos (teclado, monitor, impresora).

1.3 Virtualización

Tecnología que permite ejecutar múltiples sistemas operativos en el mismo hardware físico. Crea "máquinas virtuales" que simulan un entorno de hardware (disco, red, CPU, video).

Las máquinas que alojan VMs se llaman hipervisores. Concepto desde los años 60-70 con los mainframes IBM (S/360 Modelo 67). VMware desarrolló los primeros hipervisores modernos a fines de los 90 gracias a las extensiones IVT (Intel) y AMD-V.

TipoDescripciónEjemplos
Tipo 1 (Bare Metal)Corre directamente sobre el hardwareProxmox VE, Xen, KVM, Hyper-V, VMware ESXi
Tipo 2 (Hosted)Corre sobre un SO estándarVMware Workstation, Parallels, UTM, VirtualBox, QEMU

1.4 Características de *nix / GNU/Linux

  • Multiusuario: múltiples usuarios trabajando simultáneamente.
  • Multitarea: múltiples procesos en simultáneo.
  • Tiempo compartido: reparto del CPU entre procesos.
  • Case-sensitive: distingue mayúsculas y minúsculas.

1.5 Distribuciones: Rolling Release vs. Standard Release

  • Rolling Release: actualizaciones continuas, software más reciente, mayor riesgo de inestabilidad. Ej: Arch Linux, OpenSUSE.
  • Standard Release (Point Release): versiones estables periódicas. Ej: Debian, Ubuntu LTS, Fedora.

Jerarquías de directorios

  • Restringidos: solo modificables por administrador (/etc, /boot, /run).

Unidad 3 — Streams, Filtros, Pipes, Regex y Gestor de Paquetes EXAMEN

3.1 Standard Streams EXAMEN

⭐ Tema central del examen. Linux tiene 3 streams (flujos) predefinidos:
DescriptorNombreDefaultFunción
0stdin (Standard Input)TecladoEntrada de datos
1stdout (Standard Output)MonitorSalida normal
2stderr (Standard Error)MonitorSalida de errores (separada de stdout)

El objetivo de los 3 streams es separar entrada, salida normal y errores. Esto permite:

  • Redirigir salida a un archivo sin capturar errores.
  • Capturar y gestionar errores por separado.
  • Automatizar procesos en scripts sin intervención manual.

3.2 Redireccionamiento

SímboloFunciónEjemplo

<Lee stdin desde archivocat < archivo_ls
2>Redirige stderr a archivols /noexiste 2> error.log
2>/dev/nullDescarta errores (agujero negro)ls /noexiste 2> /dev/null
&>stdout y stderr al mismo archivo (bash 4+)ls &> salida.txt
2>&1stderr al mismo destino que stdout (compatible)ls > out 2>&1

3.3 Pipes |

El pipe | envía la salida (stdout) de un comando como entrada (stdin) del siguiente.

cat /var/log/dpkg.log | grep "installed"
cat archivo_ls | head -n 2

3.5 grep — el filtro estrella EXAMEN

Busca dentro de archivos o stdin patrones de texto o expresiones regulares.

grep [opciones] [patrón] [archivo]
  -i  : ignora mayúsculas/minúsculas
  -n  : muestra número de línea
  -r  : recursivo en directorios
  -c  : cuenta coincidencias
  -E  : usa expresiones regulares extendidas (egrep)
  -v  : invierte (líneas que NO coinciden)
⭐ Ejemplo del examen — buscar líneas que empiecen con "C": 
grep '^C' archivo.txt
El símbolo ^ indica inicio de línea.

3.6 Búsqueda: find vs locate

  • find: búsqueda exhaustiva en tiempo real. Lento pero versátil. Permite criterios: -name, -iname, -type f/d, -size, -mtime, -perm, -exec, -delete.
  • locate: ultrarrápido, usa base de datos indexada (updatedb). Parte de mlocate.

3.7 Expresiones Regulares (Regex) EXAMEN

Conjunto de reglas para definir patrones de búsqueda en texto. Se usan en grep, find, etc.

Alternación (OR)

/a|b/      → coincide con "a" o "b"

Cuantificación

SímboloSignificado

Agrupación

  • () — agrupa subexpresiones: (abc)+
  • [] — grupo de caracteres: [aeiou], [a-z0-9]
  • [^...] — negación dentro del grupo: [^aeiou]

Caracteres especiales

Unidad 4 — Redes, DNS, DHCP y SSH

4.1 Conceptos de Redes

Una red es un conjunto de dispositivos conectados para compartir recursos. Modelo cliente-servidor: el cliente genera peticiones, el servidor escucha y responde.

TipoAlcance
LAN (Local Area Network)Hogares, oficinas, edificios
WAN (Wide Area Network)Ciudades, países (ej: Internet)
PAN (Personal Area Network)Bluetooth, dispositivos personales
MAN (Metropolitan Area Network)Áreas metropolitanas

4.2 Direcciones IP

  • IPv4: 32 bits, notación decimal (4 octetos de 0-255). Ej: 192.168.1.1
  • IPv6: 128 bits, notación hexadecimal. Ej: 2001:0db8:85a3::8a2e:0370:7334

Clases de IPv4

ClaseRangoFormatoUso
A0.0.0.0 – 127.255.255.255red.host.host.hostRedes grandes
B128.0.0.0 – 191.255.255.255red.red.host.hostRedes medianas
C192.0.0.0 – 223.255.255.255red.red.red.hostRedes pequeñas
D—Multicast—
E—ReservadaInvestigación

Máscara de Red y CIDR

La máscara define qué parte es red y qué parte es host. Ejemplo: 255.255.255.0 = /24 (los primeros 24 bits son de red).

Notación CIDR: 192.168.1.1/24

Configuración de red en Debian

Archivo: /etc/network/interfaces

auto eth0
allow-hotplug eth0
iface eth0 inet static
    address 192.168.0.10
    netmask 255.255.255.0
    gateway 192.168.0.1

# o para DHCP:
iface eth0 inet dhcp

4.3 DNS — Domain Name System

Traduce nombres de dominio (www.google.com) en direcciones IP (192.168.1.1).

Jerarquía de dominios

Ejemplo: mail.ejemplo.com

  • .com = TLD (Top Level Domain)
  • ejemplo = dominio secundario
  • mail = subdominio

Tipos de servidores DNS

  1. DNS Recursivo: recibe la consulta y pregunta a otros servidores.
  2. Root DNS Server: servidor raíz, deriva al TLD.
  3. TLD Server: gestiona dominios .com, .net, .ar, etc.
  4. DNS Autoritativo: entidad final que da la IP definitiva.

Tipos de registros DNS

TipoFunción
AAsocia nombre a IPv4
AAAAAsocia nombre a IPv6
CNAMEAlias (canonical name) — dominio a dominio
MXServidores de correo
NSServidores de nombre autorizados
TXTTexto para verificaciones / políticas

4.4 DHCP — Dynamic Host Configuration Protocol

Asigna automáticamente direcciones IP, máscara, gateway y DNS a los dispositivos.

4 fases del DHCP

  1. DHCP Discover: el cliente solicita una IP (broadcast).
  2. DHCP Offer: el servidor ofrece una IP libre.
  3. DHCP Request: el cliente acepta la oferta (broadcast).
  4. DHCP ACK: el servidor confirma la asignación.

Regla mnemotécnica: D-O-R-A (Discover, Offer, Request, Acknowledge).

4.5 SSH — Secure Shell

Protocolo y programa para acceder a servidores de forma remota y segura.

  • Cliente SSH: origina la conexión.
  • Servidor SSH: recibe la conexión (paquete openssh-server).
  • Puerto default: 22 (TCP).

Archivos de configuración (/etc/ssh)

  • sshd_config: configuración del servidor SSH (port, PermitRootLogin, PasswordAuthentication, PubkeyAuthentication, AuthorizedKeysFile).
  • ssh_config: configuración del cliente.

Métodos de autenticación

  • Por contraseña: básico, vulnerable a fuerza bruta.
  • Por clave pública/privada: recomendado.
    • ssh-keygen → genera el par id_rsa (privada) e id_rsa.pub (pública).
    • ssh-copy-id usuario@host → copia la pública al servidor.
    • La clave pública se guarda en ~/.ssh/authorized_keys del servidor.
    • La clave privada NUNCA se comparte.

Uso del comando ssh

ssh host                  # con el mismo usuario actual
ssh usuario@host          # especificando usuario
exit                      # cerrar sesión SSH

5.4 Recuperación de contraseña de root desde GRUB

  1. Reiniciar la máquina.
  2. En el menú GRUB, presionar E antes que termine el contador.
  3. Localizar la línea linux /boot/vmlinuz-… ro quiet.
  4. Reemplazar ro quiet por rw init=/bin/bash.
  5. Presionar F10.
  6. Ejecutar passwd para cambiar la contraseña de root.
  7. Reiniciar con exec /sbin/init.

5.5 SUDO (SuperUser DO)

Permite ejecutar comandos con privilegios de root sin iniciar sesión como root. Características:

  • Control de acceso granular
  • Registro de actividad (auditoría)
  • Temporalidad (timestamp_timeout default 5 min)
  • Minimiza riesgos (evita uso directo de root)

Archivo /etc/sudoers

Editar SIEMPRE con visudo (verifica sintaxis).

Estructura de una regla:

alumno    ALL = (ALL:ALL) ALL
#         |     |    |    |
#       host  user grupo  comandos
  • 1° ALL: en qué máquinas se aplica la regla
  • (ALL:ALL): como qué usuario y grupo puede ejecutar
  • último ALL: qué comandos puede ejecutar

Defaults importantes

  • env_reset: resetea variables de entorno
  • mail_badpass: notifica contraseñas incorrectas
  • secure_path: PATH seguro
  • timestamp_timeout: tiempo de cache de contraseña

Alias

Host_Alias  SERVIDORES = server1, server2
User_Alias  ADMIN = alumno, jperez
Cmnd_Alias  REBOOT_CMDS = /sbin/reboot, /sbin/shutdown

Para grupos, anteponer %:

%admin ALL=(ALL) ALL

Unidad 6 — Permisos: Octal, Simbólico, Especiales y Umask

6.1 Estructura de permisos

Los permisos se dividen en 3 grupos: Usuario (Owner), Grupo (Group), Otros (Others).

Cada grupo tiene 3 permisos: r (lectura), w (escritura), x (ejecución).

-rwxr-xr--
│└┬┘└┬┘└┬┘
│ │   │   └─ otros: r--
│ │   └───── grupo: r-x
│ └───────── owner: rwx
└─────────── tipo de archivo (- regular, d directorio, l link, etc.)

6.2 Formato Octal

PermisoValor
r (read)4
w (write)2
x (execute)1
LetrasBinarioOctal
---0000
--x0011
-w-0102
-wx0113
r--1004
r-x1015
rw-1106
rwx1117
chmod 754 archivo
# 7 (rwx) owner | 5 (r-x) grupo | 4 (r--) otros

6.3 Formato Simbólico

chmod [quién][operador][permiso] archivo

quién:     u (user) | g (group) | o (others) | a (all)
operador:  + (añadir) | - (quitar) | = (establecer)
permiso:   r | w | x

Ejemplos:
chmod u+rw archivo
chmod g-w archivo
chmod a=rx archivo

6.4 Cambio de propietario

chown usuario archivo
chgrp grupo archivo
chown alumno:alumno salida.txt    # cambia ambos a la vez

6.5 Permisos Especiales

6.6 Umask

Define los permisos predeterminados al crear nuevos archivos y directorios.

  • Default para archivos: 666 (rw-rw-rw-)
  • Default para directorios: 777 (rwxrwxrwx)

Cálculo por resta

0666 - 0644 = 0022
umask 0022
# Los nuevos archivos se crearán con permisos 644 (rw-r--r--)

Cálculo por operación binaria

  1. Convertir 666 a binario: 110 110 110
  2. Convertir el deseado (644) a binario: 110 100 100 y aplicar NOT: 001 011 011
  3. Operación AND bit a bit: 110 110 110 AND 001 011 011 = 000 010 010 = octal 022

Configuración de umask

  • Sesión activa: umask 022 en la terminal.
  • Por usuario: editar ~/.bashrc o ~/.profile
  • Global: editar /etc/profile o /etc/login.defs

Unidad 7 — Filesystem (Introducción)

7.1 ¿Qué es un Filesystem?

Estructura que organiza cómo se almacenan y acceden los datos en un disco. Actúa como puente entre los datos físicos y el sistema operativo, permitiendo leer, escribir, modificar y eliminar archivos.

7.2 Tipos de Filesystems

FSCaracterísticasVentajasDesventajas
ext4Journalizado, archivos grandes, comprobación de erroresRápido y confiableNo nativo en Windows
XFSJournalizado, alta eficienciaEscalable, gran rendimiento en volúmenes grandesRedimensionado complejo
BtrfsJournalizado, snapshots, compresiónIdeal para backups y versionadoAún en optimización
NTFSCompatible con WindowsInteroperabilidad Linux/WindowsRendimiento menor en Linux

7.3 Filesystems Journalizados

Registran cada cambio en un área especial llamada journal antes de aplicarlo al sistema de archivos. Esto protege la integridad de los datos: si hay un fallo o corte de energía, el journal permite reconstruir y minimizar archivos corruptos.

Tipos de journaling

  • Journaling completo: registra datos y metadatos. Más seguro, más lento.
  • Journaling de metadatos: solo metadatos (ubicación, nombre). Más rápido, menos seguro.
  • Journaling asíncrono/sincrónico: prioriza integridad o velocidad.

7.4 Incorporación de un disco en VirtualBox

  1. Apagar la máquina y abrir Settings → Storage.
  2. Seleccionar el controlador (Controller) y "Agregar nuevo disco".
  3. Elegir Create.
  4. Definir nombre, tipo (VDI), tamaño. Dejar destildado "pre-allocate Full Size".
  5. Seleccionar el disco y confirmar con Choose y OK.
  6. Encender la máquina.

📌 Resumen ultra-condensado para repaso final:
  • Streams: stdin (0, teclado) | stdout (1, monitor) | stderr (2, monitor).
  • Redirecciones: > sobreescribe, >> añade, < entrada, 2> errores, | pipe.
  • APT: update (índice) → search (buscar) → install (instalar).
  • Stallman: GNU + GPL + software libre. Torvalds: kernel Linux (1991).
  • GNU/Linux: herramientas GNU + kernel Linux = sistema operativo completo.
  • Regex: ^ inicio, $ fin, . cualquier carácter, * cero o más, + uno o más, ? cero o una.
  • grep: filtro de búsqueda. grep '^C' archivo = líneas que empiezan con C.
